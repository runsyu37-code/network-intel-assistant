## Decision: No New Backend Needed

Frontend spec file (`F9_BACKEND_SPEC.md`) originally proposed building a new
ASP.NET Core .NET 10 backend. Decision: **skip — not worth the rebuild time.**

Reasons:
- Existing backend is feature-complete with all endpoints matching the spec
- Response shapes in the spec already match the existing API output
- System serves ~30 intranet users — .NET 10 performance gain is imperceptible
- Would require rewriting 10 controllers + all tests with zero user-facing benefit

**Outcome:** Frontend wired directly to `localhost:50680` (existing backend). Works.
