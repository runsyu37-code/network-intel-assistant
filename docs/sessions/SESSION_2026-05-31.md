### 3. Floor Plan — localStorage position fallback

**Root cause:** `GET /api/cameras` ไม่ return `position_x/y` → reload แล้ว positions reset เป็น grid

**Fix:** เมื่อ `position_x/y` เป็น null → โหลดจาก localStorage (`ssm.floorplan.positions.<floorId>`)
บันทึกลง localStorage ทุกครั้งที่ drag สำเร็จ — API values override localStorage เมื่อ backend แก้แล้ว

**ไฟล์:** `src/pages/FloorPlanPage.tsx`

### 4. BuildingDetailPage — Floor drag-to-sort (admin only)

- Grip handle (`GripVertical`) ซ้ายของแต่ละ floor card เฉพาะ admin
- HTML5 drag-and-drop: card ที่กำลัง drag fade, drop target ขอบ accent
- Order บันทึกลง localStorage (`ssm.building.floororder.<buildingId>`)
- Floor ใหม่จาก API จะ append ท้ายลำดับที่บันทึกไว้

**ไฟล์:** `src/pages/BuildingDetailPage.tsx`, `src/styles/sites.css`

### 5. Topology — Positions persist ลง API (ใหญ่สุด)

**Backend changes (branch: backend, commit: `eb8f756`):**
- `db/migration_add_site_topology_position.sql` — `ALTER TABLE sites ADD topology_x FLOAT, topology_y FLOAT`
- `sitesModel.cs` — เพิ่ม `topology_x`, `topology_y` (double?)
- `sitesController.cs` — GET return fields ใหม่ + PATCH `/api/sites/{Site_ID}/position` (admin only)

**Frontend changes:**
- `SiteApi` ใน `types.ts` — เพิ่ม `topology_x`, `topology_y`
- `hierarchy.ts` — เพิ่ม `patchSitePosition(siteId, x, y)`
- `TopologyPage.tsx` — rewrite ให้ load nodes จาก `getSites()` (real Site_IDs แทน mock) + drag → PATCH API

**Behavior:**
- Load: ใช้ `topology_x/y` จาก DB ถ้ามี, ไม่มี → radial layout
- Drag: `PATCH /api/sites/{id}/position` → persist ข้าม browser/session
- Reset Layout: คำนวณ radial ใหม่ + PATCH ทุก site
- Summary refetch ทุก 30s: อัปเดต status/count โดยไม่รีเซ็ตตำแหน่ง
- HQ node: ยัง localStorage (ไม่มี Site_ID ใน DB)

⚠️ **ต้องรัน SQL migration ก่อน Topology ใช้งานได้:**
